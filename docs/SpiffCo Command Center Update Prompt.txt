# SpiffCo Command Center — Architecture Research & Integration Task

You are a senior software engineer joining the SpiffCo Command Center project.

Your task is **NOT** to copy code from existing projects.

Instead, analyze the architecture, data flow, algorithms, and design patterns used in the following repositories and adapt those ideas into SpiffCo Command Center.

Repositories to analyze:

* https://github.com/featheredtoast/satisfactory-monitoring
* https://github.com/Jonathan-Hofmann/ficsit-remote-monitoring-dasboard
* https://github.com/TitanmasterRy/SpiffCo_Command_Center

---

# Primary Objective

Use the first two repositories as architectural references.

Determine:

* how they communicate with Ficsit Remote Monitoring
* how they request and cache data
* how they organize world objects
* how they render maps
* how factories are identified
* how buildings are grouped
* how production is calculated
* how power data is collected
* how transportation data is represented
* how refresh cycles work

Then improve upon those designs inside SpiffCo Command Center.

Do not simply recreate their implementation.

# Performance

Avoid recomputing the world every refresh.

Implement:

Spatial indexing

Caching

Incremental updates

Dirty-region recalculation

Background workers

Only recompute factories affected by changed buildings.

My recommendation for SpiffCo

If I were prioritizing the architecture, I'd take:

From featheredtoast
Historical metrics and analytics
Alerting concepts
Caching
Multi-server support
Time-series mindset
From Jonathan-Hofmann
Interactive dashboard
Responsive, game-focused UI
Direct FRM integration patterns
Storage and inventory presentation

Then I'd build what neither project has:

Factory Detection Engine (automatic campus grouping)
World Graph (relationships between mines, factories, and logistics)
Production Graph (material flow and bottleneck analysis)
SpiffCo Domain Model (treat factories as first-class objects instead of just collections of buildings)
